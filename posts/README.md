# Posts

Drafts de posts semanales antes de publicar en Substack.

## Convención
`YYYY-WNN-slug.md` — ejemplo: `2026-W17-primera-semana.md`.

## Template

Ver la plantilla base [en el repo privado](../../../machine-learning/plantillas/post-semanal.md) o duplicar uno ya publicado.

## Flujo
1. Viernes tras revisión semanal → escribir draft acá (20 min máx).
2. Domingo noche o lunes → copiar a Substack y publicar.
3. Actualizar la bitácora con el link del post publicado.

## Tono
- Honesto sobre lo que no entendí.
- Corto (200–500 palabras).
- Una conexión sorprendente > un resumen técnico.
- Identidad sobre logro: *"soy alguien que estudia en público"*, no *"publiqué un post"*.
